ISL: The Safety Layer

The LLM never writes SQL. It generates an ISL (Intelligent Structured Language) document — a constrained intent object. Table names are enum-locked to your actual schema. Every column reference is validated before compilation. The ISL is then compiled to SQL deterministically.

Zero hallucinated SQL reaches your database

Logic & Conditional Nodes

Transform graphs use Logic (Fu) and Conditional (C) nodes as first-class execution primitives. Logic nodes support ordered assignments, expression evaluation, and multiple outputs. Conditional nodes support if-else, if-elif-else, case, cond, and pattern-match branching.

Execution runs through the existing Clojure runtime — not through generated source files. No Python, no Spark, no external runtime.

Safety Guarantees

The LLM generates constrained transform GIL — a restricted subset that only allows valid transform shapes (source → logic → conditional → response → output). Topology validation enforces linear chains. Data flow validation checks that every column reference resolves to a known upstream output.

When the LLM output is invalid, a structured repair pass fires. When repair fails, deterministic fallback templates handle common patterns (order processing, customer mastering, lead scoring, invoice, eligibility).

Key Architectural Decisions

Graph Intent Language (GIL)

Structured IR between NL and graph. Constrains LLM output to valid graph operations. Deterministic compilation: same graph → same SQL every time. LLM is only in the proposal path, never in execution.

Silver as Canonical Layer

Three-layer medallion with Silver as the bidirectional contract between business demand (Gold) and source supply (Bronze). Change Gold → Silver tells you what Bronze you need.

AI in design only

AI assists proposals, explanations, and suggestions. The compiler, validator, governance gates, and execution engine are entirely deterministic. This is what makes enterprise trust possible.
